## 🖥️ Sistema Operativo

Un sistema operativo actúa como intermediario entre un usuario y el hardware de un dispositivo. Gestiona recursos esenciales como memoria, CPU, almacenamiento, etc.

### ¿Qué hace un sistema operativo?

El sistema operativo gestiona los recursos que utiliza el PC para realizar funciones básicas hacia el hardware, permitiendo una interactividad entre usuario y dispositivo.

### ¿Por qué un computador lo necesita?

Para ejecutar comandos y programas, además de administrar recursos del PC como el almacenamiento y la CPU.

### ¿Qué pasaría si un computador no tuviera sistema operativo?

Un dispositivo no podría tener una conexión entre software y hardware. Lo único que haría sería prender: no habría interactividad alguna con ningún hardware del dispositivo. Solo estaría disponible el BIOS de la placa base, lo que significa que el usuario solo podría usarlo para preparar al sistema operativo. Es necesario un SO para la utilización del hardware que posee cada dispositivo.

### ¿Es obligatorio tener un sistema operativo para programar?

No, ya que un sistema operativo está diseñado para gestionar de mejor manera y tener una interactividad entre usuario y dispositivo.

## 💻 Terminal

Una interfaz basada en texto que se usa para interactuar con los dispositivos. La terminal permite la ejecución de comandos y ver los resultados en esta. Puede ser utilizada para manejar el dispositivo, ya sea para pedir información, hacer conexiones, etc. Cada sistema operativo hoy en día contiene una terminal: Windows, Linux, macOS, etc.

### ¿Por qué muchos desarrolladores usan terminal en lugar de solo interfaces gráficas?

- Posee una mayor velocidad de búsqueda y automatización.
- Se posee un mayor control del dispositivo.
- Consume mucho menos recursos.
- No todo sistema posee una interfaz gráfica (GUI), como los servidores basados en Linux, donde solo una terminal te permite interactuar con ellos.

## 📄 README

Un README (generalmente `.md`) es un archivo de texto que contiene información fundamental sobre el software y el proyecto. Sirve como guía —un tipo de manual— donde se describe el funcionamiento y cómo utilizar el proyecto.

Puede contener información como:
- Detalles de instalación
- Descripción del programa o características
- Cómo se utiliza y qué proporciona
- Créditos de los desarrolladores

### ¿Por qué los proyectos de programación suelen tener un README.md?

Para que toda persona que quiera aprender el funcionamiento completo de un proyecto tenga acceso a su presentación, documentación, forma de uso y lo que proporciona.
